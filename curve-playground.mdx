# Market Maker Curve Playground

This interactive playground lets you design **two-sided liquidity curves** using market maker parameters. See how spread, gamma, and skew affect your bid and ask distributions in real-time.

<div id="curve-playground"></div>

---

## Understanding the Visualization

### What You're Looking At

The chart shows your **bid and ask liquidity** distributed around a central mid price:

| Element | Meaning |
|---------|---------|
| **White dashed line** (center) | Mid price - the reference point for your quotes |
| **Green curve** (left) | Bid orders - where you're buying |
| **Red curve** (right) | Ask orders - where you're selling |
| **Gray zone** (center) | Spread - the gap between your bid and ask |
| **Y-axis** | Relative liquidity density at each price |

### The Controls

| Control | Range | What It Does |
|---------|-------|--------------|
| **Spread** | 5-500 bps | Distance from mid price to your quotes. Wider = safer but less competitive. |
| **Range** | 1-50% | How far your liquidity extends beyond the spread. Wider = more price coverage. |
| **Gamma** | -5 to +5 | Curve shape. Negative = front-loaded (near spread), Positive = back-loaded (at edges), Zero = uniform. |
| **Skew** | -1 to +1 | Inventory adjustment. Negative = long (aggressive asks), Positive = short (aggressive bids). |

---

## How Gamma Shapes Your Curves

Gamma (also called exponent) controls **where your liquidity concentrates** within your range.

### Negative Gamma (e.g., -2.0)

**More liquidity near the spread, less at range edges.**

```
Density
  │
  ████                              ████
  ██████                          ██████
  ████████                      ████████
  ██████████                  ██████████
  ────────────────────────────────────────► Price
       BID        spread        ASK
```

**Use cases:**
- **Tight quoting**: Maximize fills near current price
- **High-frequency MM**: Capture small moves efficiently
- **Mean reversion**: Expect price to stay near mid

### Zero Gamma (0.0)

**Uniform distribution - equal liquidity across your entire range.**

```
Density
  │
  ██████████                  ██████████
  ██████████                  ██████████
  ██████████                  ██████████
  ██████████                  ██████████
  ────────────────────────────────────────► Price
       BID        spread        ASK
```

**Use cases:**
- **Baseline strategy**: Simple, predictable behavior
- **Uncertain markets**: No strong view on volatility
- **Testing**: Compare other strategies against this

### Positive Gamma (e.g., +2.0)

**Less liquidity near spread, more at range edges.**

```
Density
  │
  ██████████                  ██████████
  ████████                      ████████
  ██████                          ██████
  ████                              ████
  ────────────────────────────────────────► Price
       BID        spread        ASK
```

**Use cases:**
- **Volatile markets**: Capture big moves, avoid chop
- **Tail risk capture**: Position for extreme scenarios
- **Wider spreads**: Compensate with edge liquidity

## How Skew Manages Inventory

Skew adjusts **both gamma and depth asymmetrically** between your bid and ask sides. This is how you manage inventory risk.

### Negative Skew (e.g., -0.5) — You're Long

When you're holding too much inventory, you want to **sell more aggressively**:

| Side | Gamma | Depth | Effect |
|------|-------|-------|--------|
| Ask | Higher | Higher | Steeper curve, more size → fills faster |
| Bid | Lower | Lower | Flatter curve, less size → fills slower |

```
Density
  │
  ████                              ████████████
  ██████                          ██████████████
  ████████                      ████████████████
  ██████████                  ██████████████████
  ────────────────────────────────────────────────► Price
       BID (passive)    spread    ASK (aggressive)
```

### Positive Skew (e.g., +0.5) — You're Short

When you need to buy inventory, you want to **bid more aggressively**:

| Side | Gamma | Depth | Effect |
|------|-------|-------|--------|
| Bid | Higher | Higher | Steeper curve, more size → fills faster |
| Ask | Lower | Lower | Flatter curve, less size → fills slower |

```
Density
  │
  ████████████                              ████
  ██████████████                          ██████
  ████████████████                      ████████
  ██████████████████                  ██████████
  ────────────────────────────────────────────────► Price
       BID (aggressive)    spread    ASK (passive)
```

### The Skew Formula

```
askGamma = baseGamma × (1 - skew)
bidGamma = baseGamma × (1 + skew)

askDepth = baseDepth × (1 - skew)
bidDepth = baseDepth × (1 + skew)
```

## Real-World Examples

### Example 1: Neutral Market Maker

*"I want symmetric quotes with moderate concentration near spread."*

**Settings:**
- Spread: **30 bps**
- Range: **5%**
- Gamma: **-1.0** (front-loaded)
- Skew: **0** (neutral)

**Result:** Equal bid/ask curves, both concentrated near the spread. Classic two-sided MM.

### Example 2: Inventory Rebalancing

*"I'm long 500 SOL and need to reduce exposure."*

**Settings:**
- Spread: **25 bps**
- Range: **8%**
- Gamma: **1.5**
- Skew: **-0.4** (long, want to sell)

**Result:** Ask side has 40% more depth and steeper gamma. You'll sell faster while still maintaining a bid.

### Example 3: Volatile Market

*"High volatility expected, want to capture big moves."*

**Settings:**
- Spread: **100 bps** (wider for safety)
- Range: **15%** (extended range)
- Gamma: **+2.0** (back-loaded)
- Skew: **0** (neutral)

**Result:** Most liquidity at range edges. Small fills on normal moves, big fills on breakouts.

### Example 4: Tight Scalping

*"Low volatility, want to capture every tick."*

**Settings:**
- Spread: **10 bps** (tight)
- Range: **2%** (narrow)
- Gamma: **-2.0** (heavily front-loaded)
- Skew: **0** (neutral)

**Result:** Almost all liquidity right at the spread. Maximum fill rate in quiet markets.

## From Playground to Code

### High-Level SDK (MM-Friendly)

```typescript
// The SDK computes per-side parameters from your MM inputs
const orders = await market.placeMMOrders({
  midPrice: 150_000_000n,     // $150 reference
  spreadBps: 30,              // 30 bps each side
  rangeWidth: 0.05,           // 5% range beyond spread
  gamma: 1.5,                 // Base curve shape
  skew: -0.3,                 // Long, want to sell
  maxBase: 10_000_000_000n,   // 10 SOL total
  maxQuote: 1_500_000_000n,   // 1500 USDC total
});

// This creates TWO orders:
// BID: gamma=1.95, depth=130% of base
// ASK: gamma=1.05, depth=70% of base
```

### Low-Level SDK (Direct Control)

```typescript
// Or place orders directly with raw parameters
const bidOrder = await market.placeClmmPosition({
  pMin: 142_500_000n,         // $142.50
  pMax: 149_550_000n,         // $149.55 (spread edge)
  exponent: 1950,             // 1.95 (scaled by 1000)
  curveShift: 0,
  maxBase: 13_000_000_000n,   // 13 SOL (130% of 10)
  maxQuote: 0n,
  side: 0,                    // Bid
});

const askOrder = await market.placeClmmPosition({
  pMin: 150_450_000n,         // $150.45 (spread edge)
  pMax: 157_500_000n,         // $157.50
  exponent: 1050,             // 1.05 (scaled by 1000)
  curveShift: 0,
  maxBase: 7_000_000_000n,    // 7 SOL (70% of 10)
  maxQuote: 0n,
  side: 1,                    // Ask
});
```

<Note>
**Parameter scaling**: On-chain, exponent and curveShift are i32 scaled by 1000. So gamma 1.5 → 1500, curveShift -0.2 → -200.
</Note>

## Parameter Reference

| Parameter | On-Chain Type | Scale | Range | Description |
|-----------|---------------|-------|-------|-------------|
| `exponent` | `i32` | ×1000 | -5000 to +5000 | Gamma/curve shape |
| `curve_shift` | `i32` | ×1000 | -1000 to +1000 | Horizontal shift within range |
| `p_min` | `u64` | Native | - | Lower price bound |
| `p_max` | `u64` | Native | - | Upper price bound |
| `amount` | `u64` | Native | - | Liquidity amount |
